Maven依赖

依赖传递

引用的架包如果包含其他引用会自动将其引用的架包引用进本项目
举例 项目A 项目B依赖于项目A 项目C依赖于项目B 则项目C会将A自动引入自己的依赖库中 如果不想让其引入进来可以使用exclusion标签将其排除

依赖冲突

解决冲突有两种方式

  • 短路优先
    A->B->C->X A->D->X会解析第二种方式
  • 先声明先优先 （路径相同的情形）
    架包声明在前，则选择在前的依赖

聚合和继承

  • 聚合

    使用modules标签 module指定项目位置

  • 继承

    父项目使用dependencyManagement标签
    子项目使用parent标签

    Maven使用插件

    可以直接使用插件运行web项目
    即添加plugin

Maven pom.xml解析

发表于 2018-07-23 | 分类于 Java

pom.xml 文件解析

标签的含义

  • modelVersion 指定当前pom的版本
  • groupId实际项目的名称

    公司名称的反写+项目名称

  • artifactId模块的名称

    项目名称+模块名称

  • version 版本号

    三个数字表示 大版本号 分支版本号 小版本号
    0.0.1
    snapshot快照版本
    alpha 内部测试版本
    beta公测版本
    Release稳定版本
    GA正式发布版本

  • packing 打包方式

    默认方式jar

  • dependencies依赖列

    dependency依赖项
    scope依赖范围 对应的maven生命周期
    optional 设置依赖是否可选，决定子项目是否显示依赖此架包
    exclusion 排除依赖传递列表

  • dependencyManagement依赖管理

    不会引用到到实际项目，给子模块继承用

  • build 构建行为

    plugins插件列表
    plugin插件

  • parent用于子模块对父模块的继承
  • modules指定多个模块进行编译
  • name项目描述名
  • ur 项目地址
  • description项目描述
  • developers开发人员信息
  • license许可证信息
  • organization组织信息

Maven常用命令

发表于 2018-07-22 | 分类于 Java

Maven重用命令

  • 常用构建命令

    mvn -v 查看版本
    mvn -compile 编译
    mvn -test 测试
    mvn -package打包
    clean 删除target
    install 将jar包安装到本地仓库

  • 自动构建目录骨架

    archetype插件 用于创建符合maven规定的目录骨架
    mvn archetype:generate
    mvn archetype:generate -DgroupId=x.x.x -DartifactId=x.x.x -Dversion=x.x.x -Dpackage=x.x.x

  • Maven中的坐标和仓库

    构件通过坐标作为其唯一的标识
    仓库分为本地仓库和远程仓库
    镜像仓库 mirror

更改仓库位置

maven 生命周期

  • clean

    pre-clean 执行清理前的工作 clean 清理上一次构建生成的所有文件 post-clean 执行清理后的工作

  • default 构建项目（最核心）

    compile test package install

  • site生成项目站点

    pre-site生成站点前要完成的工作 site生成项目的站点文档 post-site在生成项目站点后要完成的工作 site-deploy 发布生成的站点到服务器上
